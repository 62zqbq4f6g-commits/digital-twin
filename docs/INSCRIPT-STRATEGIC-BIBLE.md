They're building action. We're building understanding.

# PART II: NON-NEGOTIABLES

## 4. What We Will Never Compromise

### 4.1 Privacy Is the Product

- **User data is never used to train AI models.** Enterprise LLM tier only.
- **Delete means delete.** No soft-delete gray areas for user-initiated deletion.
- **Encryption by default.** Notes encrypted before leaving the device.
- **Transparency always.** Users can see exactly what Inscript knows about them.

*If a feature requires compromising privacy, we don't build it.*

### 4.2 Portability Is Power

- **Users own their memory.** Full export in human-readable format, always.
- **No lock-in through obscurity.** We publish our memory format spec.
- **Interoperability is a feature.** We want users to take their memory to other AIs.

*The memory that travels with you is more valuable than the memory that traps you.*

### 4.3 Trust Is Earned Daily

- **We don't announce what we don't have.** No vaporware.
- **We admit when we're wrong.** Publicly.
- **We ship what we promise.** On time.

*Every interaction either builds or erodes trust. There is no neutral.*

### 4.4 People Over Productivity

- **We remember people, not just tasks.** Marcus matters more than "buy milk."
- **Relationships are the core data model.** Everything connects to entities.
- **Emotional context is valued.** Sentiment, not just facts.

*We're building a mirror, not a task manager.*

# PART VIII: CULTURE & PRINCIPLES

## 12. How We Make Decisions

### 12.1 Product Principles

1. **Trust over growth.** We will not compromise privacy for metrics.

2. **Depth over breadth.** Better to know 10 things deeply than 100 things shallowly.

3. **Show, don't tell.** Users should see Inscript learning, not just read about it.

4. **People over productivity.** Relationships matter more than tasks.

5. **Open over closed.** Portability builds trust; lock-in erodes it.

### 12.2 Decision Framework

When making product decisions, ask:

1. **Does this build trust?** If no, reconsider.
2. **Does this deepen memory?** If no, deprioritize.
3. **Does this move us toward platform?** If no, question timing.
4. **Can users export this data?** If no, redesign.
5. **Would we be proud to explain this publicly?** If no, don't do it.

### 12.3 What We Say No To

- Features that require server-side access to unencrypted user data
- Growth hacks that erode trust (dark patterns, manipulative notifications)
- Integrations that don't support two-way data flow
- Enterprise features before consumer love
- Anything that makes users feel surveilled rather than understood
