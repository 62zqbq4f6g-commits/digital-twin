PHOTO FROM CAMERA TOO LARGE - NEEDS COMPRESSION

iPhone camera photos are 3-5MB. API chokes on large images.

FIX: Compress all images before upload

1. Add compression function that:
   - Resizes to max 1024px width/height
   - Compresses to JPEG quality 0.8
   - Target: under 500KB

2. Apply compression before base64 conversion:
   - User selects image → compress → convert to base64 → send to API

3. Show feedback during compression:
   - "Compressing image..." toast or indicator

Code:
function compressImage(file, maxSize = 1024, quality = 0.8) {
  return new Promise((resolve) => {
    const img = new Image();
    img.onload = () => {
      const canvas = document.createElement('canvas');
      let width = img.width;
      let height = img.height;
      
      if (width > height && width > maxSize) {
        height = (height * maxSize) / width;
        width = maxSize;
      } else if (height > maxSize) {
        width = (width * maxSize) / height;
        height = maxSize;
      }
      
      canvas.width = width;
      canvas.height = height;
      canvas.getContext('2d').drawImage(img, 0, 0, width, height);
      canvas.toBlob(resolve, 'image/jpeg', quality);
    };
    img.src = URL.createObjectURL(file);
  });
}
